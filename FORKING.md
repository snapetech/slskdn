# Forking slskdN

slskdN is an unofficial fork of slskd. This repository keeps upstream attribution in source headers, `LICENSE`, and `NOTICE`, and identifies the running client as slskdN with a distinct Soulseek minor version.

If you publish a fork of slskdN, keep the upstream slskd notices intact, keep slskdN notices intact for slskdN-specific work, and use your own project name and client identity for any distributed modified version. Do not present your build as upstream slskd or as an official slskdN release unless you are authorized to do so.

Practical checklist for third-party forks:

- Choose a distinct product name, package name, support URL, and visible UI/API branding.
- Preserve AGPL licensing, upstream attribution, slskdN attribution, and all existing copyright headers.
- Review the upstream slskd Additional Terms before distribution and make sure your login/client identity remains distinguishable.
- Document any new behavioral differences so users can tell which project they are running.
- Target your own repository, issue tracker, release tags, package feeds, and container images.
